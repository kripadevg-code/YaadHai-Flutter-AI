<parameter name="CodeContent"># YaadHai ✨

**AI-powered Learning, Memory & Mastery Platform**

Scan your study material, get instant AI-generated summaries, flashcards, quizzes, and a spaced-repetition schedule — all offline-first with server-side Gemini AI.

---

## ✨ Features

- 📷 **Scan & Learn** — Camera or gallery scan → OCR → AI Study Pack in seconds
- 🧠 **AI Study Packs** — Summary, key concepts, flashcards, and quiz generated by Gemini
- 🌳 **Visual Knowledge Tree** — Interactive graph of concept relationships
- 📊 **Learning Insights** — Charts showing concept complexity, retention curves, and load split
- 🗂️ **Spaced Repetition** — Automatic revision scheduling based on mastery level
- 🏆 **Mastery Tracking** — Per-subject and per-concept progress tracking
- 📥 **PDF Export** — Download full study packs as colorful, well-designed PDFs
- 🔖 **Bookmark & Share** — Bookmark packs for quick access; share via native share sheet
- 🌙 **Dark / Light Mode** — Adaptive theming throughout
- 🔐 **Auth** — Google Sign-In via Supabase Auth

## 🚀 Quick Start

### 1. Clone the repo

```bash
git clone https://github.com/kripadevg-code/YaadHai-Flutter-AI.git
cd YaadHai-Flutter-AI
```

### 2. Install the correct Flutter version

```bash
fvm install        # reads .fvmrc and installs the pinned version
fvm flutter doctor # verify your environment
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in your values:

```dotenv
# ── Supabase ──────────────────────────────────────────────────────────────────
SUPABASE_URL=https://<your-project-ref>.supabase.co
SUPABASE_ANON_KEY=<your-anon-key>

# ── Google Sign-In ────────────────────────────────────────────────────────────
# Use the WEB client ID from Google Cloud Console
# (Supabase requires this for token exchange even on Android/iOS)
GOOGLE_WEB_CLIENT_ID=<your-web-client-id>.apps.googleusercontent.com
```

### 4. Install packages

```bash
fvm flutter pub get
```

### 5. Run the app

```bash
fvm flutter run
```

## ☁️ Supabase Setup

All AI inference is handled **server-side** via Supabase Edge Functions. The Gemini API key never touches the Flutter app.

### a) Create a Supabase project

Go to [supabase.com](https://supabase.com) → New Project.

### b) Enable Google Auth

1. Supabase Dashboard → **Authentication → Providers → Google**
2. Add your **Google Web Client ID** and **Client Secret**
3. Copy the **Callback URL** shown and add it in Google Cloud Console → OAuth credentials → Authorized redirect URIs

### c) Deploy Edge Functions

```bash
# Log in to Supabase CLI
supabase login

# Link to your project
supabase link --project-ref <your-project-ref>

# Deploy both functions
supabase functions deploy gemini-generate
supabase functions deploy gemini-vision
```

### d) Set the Gemini secret

```bash
supabase secrets set GEMINI_API_KEY=<your-gemini-api-key>
```

Get a Gemini key at [aistudio.google.com](https://aistudio.google.com/apikey).

## 📱 Platform Configuration

### Android

1. In [Google Cloud Console](https://console.cloud.google.com/) → **APIs & Services → Credentials**, create an **Android** OAuth client:
   - Package name: `com.yaadhai.yaad_hai`
   - SHA-1: run `./gradlew signingReport` in `android/` (debug fingerprint)

2. No `google-services.json` is needed — auth flows through Supabase.

### iOS

1. In Google Cloud Console, create an **iOS** OAuth client:
   - Bundle ID: `com.yaadhai.yaadHai`
2. Add the reversed client ID as a URL scheme in `ios/Runner/Info.plist`:
   ```xml
   <key>CFBundleURLTypes</key>
   <array>
     <dict>
       <key>CFBundleURLSchemes</key>
       <array>
         <string>com.googleusercontent.apps.YOUR_IOS_CLIENT_ID</string>
       </array>
     </dict>
   </array>
   ```

## 🗄️ Database

The app uses **Drift** (SQLite) for fully offline-first local storage. No migrations need to be run manually — Drift handles schema upgrades automatically on first launch.
